# Plans (slices)

One file per **bounded slice** of work (a milestone, a week, a feature chunk)—not an eternal master spec.

## Current status

- Active plan: [`2026-04-21-v1-foundation-plan.md`](2026-04-21-v1-foundation-plan.md)
- Repository is currently docs-first (no runtime code yet), so plan files are the execution contract.

## When to add a plan

- The slice touches several files or subsystems, or
- You want checkboxes and verification steps so execution stays on track.

## When to finish a plan

- Mark tasks done, or
- Move the file to `archive/` with a date prefix if you want history, or
- Delete it if the value is only in git history.

## Suggested plan shape

1. **Goal** — one sentence, observable “done.”
2. **Non-goals** for this slice.
3. **Files** — create/modify/test paths.
4. **Tasks** — checkboxes; each step verifiable in minutes.
5. **Verification** — exact command(s) and what “pass” looks like.
